각 에이전트는 두 가지 모드를 지원한다.

- **질문 생성 모드**: 그 사람 특유의 스타일·카테고리로 테스트 질문 N개 생성.
- **피드백 모드**: 질문 + 봇 응답 묶음을 주면 그 사람의 평가 축과 말투로 피드백 + 집계 JSON 출력.

## 봇 개선 워크플로우

봇 라벨은 호출 시 주어지는 대로 따른다(2주차 시트는 A=통합/B=원리/C=정밀, `syste-prompt-ver/_experiment/`는 A_원리/B_정밀정보/C_선배/D_통합v5/E_달인_블렌드).

1. **질문 생성** — 6개 에이전트를 병렬로 호출해 각자 질문 N개씩 생성. 페르소나별 주력 카테고리가 달라 자연스럽게 커버리지가 분산된다.
2. **응답 수집** — 생성된 질문을 각 봇(시스템 프롬프트 버전)에 입력해 응답 수집. 기존 `syste-prompt-ver/_experiment/results_*.json` 방식 재사용.
3. **페르소나 피드백** — 질문+응답 묶음을 6개 에이전트에 병렬 전달. 각자 피드백 모드로 평가.
4. **집계·개선** — 6개의 집계 JSON을 모아 봇별 반복 약점 도출 → 시스템 프롬프트/RAG 수정 → 같은 질문으로 재평가(회귀 테스트).

호출 예시 (Agent 도구).

```
subagent_type: redteam-shineunbi
prompt: |
  피드백 모드. 아래 질문과 4개 봇(A~D)의 응답을 평가해줘.
  [질문] ...
  [A 응답] ...
  [B 응답] ...
  ...
```

## 주의사항

- 새로 추가된 에이전트는 **새 세션부터** `subagent_type`으로 인식된다.
- 페르소나 피드백은 그 사람의 실제 피드백 데이터(30건 안팎)에서 추출한 관점의 재현이며, 실제 본인의 의견이 아니다. 외부 공유 시 이 점을 명시할 것.
- 사실 정확성(조화연 축)의 최종 판정은 사람 검증 필요 — 에이전트는 규정·공문의 실제 내용을 모르는 상태에서 "확인 필요"를 표시하는 역할까지만 신뢰할 것.
