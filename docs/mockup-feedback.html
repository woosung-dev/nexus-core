피드백 사유 기능 — 디자인 목업

기존 사이트에 새로 추가되는 요소는 NEW 로 표시했어요. 실제 적용 시 디자인 토큰/간격은 코드베이스 톤에 맞춰 조정.

1. 클라이언트 — 부정 피드백 (👎) 클릭 직후

👎 누르는 즉시 feedback=down 기록 + 사유 폼이 인라인으로 자동 펼침. 카테고리 칩 + 자유 텍스트 + 건너뛰기.

부정확함 도움 안 됨 근거 부족 너무 김 부적절 기타

2. 클라이언트 — 긍정 피드백 (👍) 클릭 직후

👍 클릭 시 즉시 feedback=up 기록. 사유 폼은 자동 안 열리고, 옆에 작은 "+ 사유 추가" 링크만 표시.

축복앱AI · 어시스턴트
안녕하세요! 축복식 참석에 대해 안내드릴게요. 가족 외에도 친한 친구분들 모두 환영합니다. 다만 사전에 인원수를 알려주시면 좌석 준비에 도움이 됩니다.
NEW+ 사유 추가

↑ "+ 사유 추가" 클릭 시 동일한 인라인 폼이 펼쳐짐 (긍정용 칩으로: 정확함, 도움 됨, 친절함, 명확함, 기타)

3. 관리자 — 피드백 포커스 탭 (사유 컬럼 추가)

기존 컬럼 옆에 "사유" 컬럼을 추가. 카테고리는 작은 배지로, 자유 텍스트는 1~2줄 미리보기 + hover 시 전체.
