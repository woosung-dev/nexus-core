## 2. 진단 — 왜 점수가 낮은가

### 진단 A. 시스템 프롬프트가 "정답"을 막고 있다 (가장 큰 원인)
레드팀이 원한 콘텐츠 상당수는 **이미 규정집(RAG) 안에 정확히 존재**한다. 그런데 현재 프롬프트의 다음 규칙들이 그 답을 누르고 있다.

| 현재 프롬프트 규칙 | 부작용 | 레드팀 근거 |
| --- | --- | --- |
| "규정집 문장을 길게 인용하지 않음 / 사람 말투로 풀어서" | 원리·말씀·구체 기준이 희석됨 | 신앙·원리 부족 79건, "말씀 인용 원함"(#246) |
| 금지 단어 `반드시·필수·금지·자격 됩니다/없습니다` | 규정집에 명시된 **금지·기준**조차 단정 못 함 | 가정출발 금지행동 회피(#72), 예복(#58) |
| "한 번에 교리·규정을 쏟아내지 않음" | 충실도·깊이 부족 | 71건 |
| "단정 금지 / 담당자 확인 우선" | "담당자에게 확인하세요" 기계적 남발 | 자유서술 직접 지적, 동문서답 21건 |

**대표 사례 #72** — 질문 "가정출발 전까지 금지된 행동(스킨십 정도, 혼숙)이 있나?"
→ 봇 답변: *"금지의 목록으로 채우기보다… 담당 목회자와 상담하며 지혜를 구하는 것이 가장 좋습니다."* (회피)
→ 그러나 규정집은 **명확히 답한다**: 성별기간(40일) 중에는 *"친 형제자매 관계를 넘지 않도록"* 지도하며, *"성별기간 중에 부부관계를 맺으면 성별실패"*. 신체접촉의 단계도 탈선/중도/경도로 분류(아래 4장 참조). → 답을 알면서 못 한 것.

### 진단 B. RAG 적재 위생 문제 (검증 필요)
`backend/uploads/`에 규정집이 `.docx`·`.pdf`·변환 `.docx.md`(926KB, **내용은 깨끗함**)·`.txt`로 중복 산재. 그중 **`db9c3fe3_축복항정 국제 규정집.docx`는 10바이트짜리 깨진 파일**(파일명도 "축복**항**정" 오타). 운영 봇 File Search에 어떤 file_id가 실제 색인됐는지 확인이 필요하다. 클린 변환본이 잘 색인돼 있다면 RAG 본문 품질 자체는 양호하다(아래 4장 추출 결과가 증거).

### 진단 C. 규정집 밖 영역(운영·실무)의 진짜 공백
"현실적 조언 부족"(101건)의 상당수는 교리/규정이 아니라 **현장 운영·실무**다. 규정집에 **없는** 것 — B4U(블레싱포유) 매칭 사이트 사용·연락·거절 절차, 교류~약혼 평균 소요기간, 축복헌금 환불 기준, 교육 이수 연한·강좌명, 천애축승(참어머님 소천 후 진행), 정신과 치료자 대상 정책. 이 영역은 **별도 자료가 있어야** 채울 수 있다(임의 생성 금지).

## 3. 개선 레버 (우선순위 순)

### 레버 1 — 시스템 프롬프트 재조정 (최우선, 비용 최소)
"안심·경청" 기조와 **§7 안전·위기 규칙은 그대로 둔다.** 억제 규칙만 조건부로 완화한다.

**B1. 금지 단어 규칙 정밀화**
- (유지) 개인의 자격·미래를 단정하는 표현은 계속 피함 — "자격 됩니다/없습니다", "무조건 괜찮습니다".
- (신설 예외) **규정집에 명시된 행정 기준·금지·절차는 규정집 표현 그대로 명확히 전달**한다.
  - 예: *"성별기간 40일 동안 부부관계를 맺으면 규정상 '성별실패'로 분류됩니다."* / *"탈선 문제가 있는 경우 규정상 은사 과정을 거쳐야 축복 대상이 됩니다."* — 이때의 단정은 개인 단죄가 아니라 규정 인용이므로 허용.

**B2. 신앙·원리 관점 의무화**
- 행정·실무 답변에도 **축복의 원리적 의미를 1\~2문장** 곁들인다. 가능하면 천성경/참부모님 말씀을 짧게 인용(예: *"축복이란 복을 비는 것이며, 그 복 중 사랑의 복이 가장 귀하다고 하셨어요."*).
- (변경) "규정집 길게 인용 금지" → "**핵심 근거는 짧게 인용해도 됨**. 길게 늘어놓지만 않으면 됨."

**B3. "담당자 확인" 트리거 한정**
- 규정집에서 확인되는 사실은 **먼저 답한다.** "담당자/목회자 확인"은 (a) 개인별 적용이 실제로 갈릴 때, (b) 규정집에 없는 사항, (c) 비용·일정 등 현장 변동 항목에만 덧붙인다. 모든 답에 기계적으로 붙이지 않는다.

**B4. 사용자 상황 단정 금지 → 분기**
- 교류 여부 등 상황을 멋대로 가정하지 않는다. 답이 갈리면 *"교류 중이신 경우 / 아직 아닌 경우"*로 분기하거나 한 줄로 되묻는다. (#113 근거)

**B5. 응답 구조(행정/실무) 보강**
- `결론 → 규정 근거(+말씀 1문장) → 구체 절차·체크리스트 → 개인 적용 시 확인할 점 → 다음 한 걸음`. 핵심을 먼저, 깊이는 질문의 깊이에 비례하게.

> **#72 재작성 예시 (after)**
> 좋은 질문이에요. 가정출발 전 성별기간은 규정상 분명한 기준이 있어요.
> - 성별기간은 축복식 다음날부터 **40일**이고, 이 기간은 부부가 되기 전 하늘 앞에 정성을 들이는 때로 둡니다.
> - 규정에서는 이 기간을 *"친 형제자매 관계를 넘지 않도록"* 지키라고 안내해요. 즉 부부관계는 물론, 연인 사이의 신체접촉(키스·애무 등)도 이 기간에는 두지 않는 것이 기준이에요. 성별기간 중 부부관계를 맺으면 규정상 '성별실패'로 다뤄집니다.
> - 다만 구체적인 상황 적용은 담당 목회자와 함께 정리하시면 좋아요.
> 무엇을 하지 말지의 목록이라기보다, 두 사람이 축복의 가치를 지키며 출발을 준비하는 시간으로 받아들이시면 됩니다.

### 레버 2 — RAG 위생 + 보강
- **(a) 적재 검증** — `GET /admin/bots/{id}/documents`로 운영 봇 색인 목록 확인 → 10바이트 깨진 파일·중복 제거, **클린 변환본(규정집)을 단일본으로 정돈.**
- **(b) 분할 색인 검토** — 122쪽 혼합 문서를 섹션(말씀 / 가정출발 / 성적문제 / 영인·은사 / 정리 / 식순) 단위로 나눠 적재하면 검색 정확도 향상 기대. 재색인 후 A/B 비교.
- **(c) 신앙 콘텐츠 보강** — 사용자 제공 예정인 원리강론·천성경·참부모님 말씀 원문을 정제해 추가. **천성경 참가정편 일부는 이미 규정집에 있으므로 중복은 피하고 부족분 위주.**

### 레버 3 — FAQ 오버라이드 핀포인트 (정확도 보증)
`faqs`(pgvector)는 RAG보다 먼저 체크되고 유사도 0.85↑면 그대로 반환된다. **고빈도 + 저만족 질문**에 검수된 정답을 박는다. 각 답은 `규정 근거 + 신앙적 의미 1문장 + 구체 절차 + (필요시) 확인 필요 항목` 구조.

후보(규정집 근거 확보분):
1. **"가정출발 전 금지 행동/성별기간"** — 위 #72 재작성 예시.
2. **"축복 은사가 뭔가요"** — 규정 정의: *"축복 후 혈통을 잃은 사람이 하늘의 특별한 용서의 은사를 받아 다시금 축복으로 복귀하는 것."*
3. **"배우자 성화 후 재축복 가능?"** — 조건부 정답: **가정출발(3일행사) 전** 성화 → 축복 정리 후 지상 재축복 가능 / **가정출발 후** 성화 → 위로축복(지상에서만의 부부) 또는 사별 배우자와 영육계·영인축복.
4. **"축복 절차 전체 흐름"** — 후보자 자격 → 매칭(천일국/부모) → 성주식 → 축복식 → 탕감봉 → 성별 40일 → 3일행사.
5. **"축복 준비 연령/자격"** — 부모매칭 만20세(만17세부터 양가 합의 시), 천일국매칭 만20~30세 + 후보자 자격(아래 4장).

## 6. 실행 로드맵 & 검증

**Phase A (현재)** — 본 문서 검토. 신앙 콘텐츠·금지기준 표현은 **교리 검수 필요**(목회자/담당자). 사용자 제공 예정 신앙 원문, ⚠️확인 필요 항목 자료 수집.

**Phase B (승인 후)**
1. 시스템 프롬프트 v5 작성(레버1) → `syste-prompt-ver/`에 신규(기존 v4 보존) → 어드민 UI 또는 `_seed_bots.sh` 반영.

4. 시스템 에러(#0) 별도 수정.
